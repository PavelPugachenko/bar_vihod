# Бар "Выход" - Django

## Запуск локально

```bash
cd /Users/pavel/bar_vihod
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
# В .env можно оставить DJANGO_DEBUG=True для локального режима
python3 manage.py runserver
```

Открыть: `http://127.0.0.1:8000`

## Режим без БД

Проект работает в режиме `TG-only`: заявки не пишутся в базу данных, а сразу отправляются в Telegram.
`db.sqlite3` не используется.

## Telegram для уведомлений о брони

Если переменные не указаны, форма покажет ошибку настройки Telegram и заявка не отправится.

```bash
export TELEGRAM_BOT_TOKEN="123456:ABCDEF..."
export TELEGRAM_CHAT_ID="-1001234567890"
```

После этого новые заявки с сайта будут отправляться менеджеру в Telegram.

## Что реализовано

- одностраничный сайт с меню, фото гостей и командой;
- меню перенесено в текстовом формате с референса;
- форма брони (имя, телефон, дата, время);
- валидация окна бронирования: с `14:00` до `02:00`;
- отправка заявки напрямую в Telegram без сохранения в БД.
